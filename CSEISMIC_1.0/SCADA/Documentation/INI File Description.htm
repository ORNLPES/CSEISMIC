INI Files

All sections absolutely have to be included, even if no key/value pairs are present. All keys must be unique within a given INI file, even between sections.

DeviceInfo

This section contains metadata regarding the device. None of this information regards data that needs to be passed from slave to master or vice versa.

 

Name – Unique name of the device as a string.

DeviceType – Unique type. Must be from the following list (case sensitive):

·      Storage

·      PV

·      Generator

·      Relay

·      MainSwitch

·      Wind

IP Address – The IP Address of the Target as a string.

Port – The port on which the Target will connect for Modbus operations.

Activations

This section contains all information regarding the activation and deactivation of the IED.

 

Activate – For most devices this puts the device into a state in which it can operate freely. That is, receive setpoints and produce/supply power. For relays this corresponds to the relay closing.

Deactivate – This puts the device into a state where it does not produce/supply power. For relays this corresponds to the relay closing.

Control – This is a Coil used by the master to direct the slave.

Timeout – This is an Input Register used by the slave to tell the master how long the operation can take without an error occurring. This value is in seconds.

Status – This is a Discrete Input used by the slave to notify the master whether or not the device is in the specific state. For example after the activate command, the device will assert this value only after being ready to produce power/has closed the relay. The status addresses should be contiguous.

Watchdog – This is a Holding Register that the master writes to. If the value hasn’t changed within 10 seconds the device needs to deactivate.

Parameters

This section includes parameters that are specific to individual device types and are outside of setpoints and measurements.

 

Each Parameter is specified individually. But in general data of types larger than 16 bits will be placed in sequential addresses.

GenerationModeSelect – This Coil determines the generation mode of the IED. False = P/Q, True = V/f.

StorageCapacity – The maximum energy that can be stored in Kilowatt Hours. This is stored as a UINT16 Input Register with optional scaling.

HighVoltageLimit – This is the value in Volts that will trigger an overvoltage Fault. This is a Holding Register storing a single UINT16 with optional scaling.

LowVoltageLimit - This is the value in Volts that will trigger an undervoltage Fault. This is a Holding Register storing a single UINT16 with optional scaling.

ForwardCurrentLimit - This is the value in Volts that will trigger an overcurrent Fault. That is, that there is a fault in the microgrid and too much current is being pulled off of the grid. This is a Holding Register storing a single UINT16 with optional scaling.

ReverseCurrentLimit - This is the value in Volts that will trigger an overcurrent Fault. That is, that there is a fault in the grid and too much current is being pulled off of the microgrid. This is a Holding Register storing a single UINT16 with optional scaling.

HighFrequencyLimit - This is the value in Hertz that will trigger an overfrequency Fault. This is a Holding Register storing a single UINT16 with optional scaling.

LowFrequencyLimit - This is the value in Hertz that will trigger an underfrequency Fault. This is a Holding Register storing a single UINT16 with optional scaling.

TotalCapacity – This is the total possible generation capacity of a renewable resource in Kilowatts. This is an Input Register storing a single UINT16 with optional scaling.

Setpoints

This section contains all setpoint addresses for a given IED. Each value is stored in a single Holding Register with optional scaling.

 

Real – This is the Real Power setpoint in Watts. This is only used in P/Q mode.

Reactive – This is the Reactive Power setpoint in VArs. This is only used in P/Q mode.

Voltage – This is the target voltage in Volts. This is only used in V/f mode.

Frequency – This is the target frequency in Hertz. This is only used in V/f mode.

Measurements

This section contains all measurements returned by the IED. Each value is stored in a single Input Register with optional scaling.

 

Current – Represented in Amps

Voltage – Represented in Volts.

RealPower – Represented in Watts.

ReactivePower – Represented in VArs.

Scaling

This section contains all of the scaling values for keys within the INI file. The value given is the multiplier in which a UINT16 is converted to a double. For example a scaling value of 0.01 would convert a register value from 43225 to an actual value of 432.25.
